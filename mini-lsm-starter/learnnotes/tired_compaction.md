## space amplification trigger 

防止空间过度膨大，计算的是除了最后一层的总大小/最后一层大小 的比值
如果比值太大，说明新累计的数据非常多，需要进行合并，把新的数据合并为一个新的tier

触发频率相对较低，但是每次都需要重写一次数据，存在比较高的写放大

## size ratio trigger

如果一个tier比他前面的（也就是更新的）tier明显的大的时候，就需要把前面这些小的tier和它合并为一个更大的tier
但是不是符合条件的全都合并，是从最上层逐层向下扫描，仅合并第一个符合条件的情况

这种方法也是控制写放大，防止出现总是合并为tier的情况，每次都是只合并一小批tier

频率相对较高，但是触发的时候仅仅合并几个tier

## reduce sorted run

如果前面两个条件都不满足，需要使用这个来防止tier过多
